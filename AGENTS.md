# vaultweaver

Kuberentes operator that syncs Vault auth roles and policies via CRDs (`security.platform.io/v1alpha1`). Multi-group Kubebuilder project with a single `security` group.

## Project layout

```
cmd/main.go                            Entrypoint — registers controllers, inits Vault client
api/security/v1alpha1/                 CRD types: VaultKubernetesRoleBinding, VaultPolicy
internal/controller/security/          Reconciliation logic for both CRDs
internal/vault/                        Vault client interface + impl + MockClient
test/e2e/                              E2E tests (build tag: e2e)
dist/chart/                            Helm chart
```

## Never edit (auto-generated)

- `config/crd/bases/*.yaml`, `config/rbac/role.yaml` — regenerated by `make manifests`
- `**/zz_generated.*.go` — regenerated by `make generate`
- `PROJECT` — kubebuilder metadata

Do not delete `// +kubebuilder:scaffold:*` comments — CLI injects code at these markers.

## Key commands

| Action | Command |
|---|---|
| Regen CRDs/RBAC | `make manifests` |
| Regen DeepCopy | `make generate` |
| Lint | `make lint` (uses golangci-lint v2 + logcheck plugin from `.custom-gcl.yml`) |
| Auto-fix lint | `make lint-fix` |
| Unit tests (envtest) | `make test` — excludes e2e (`grep -v /e2e`), requires `make setup-envtest` first |
| Single-package test | `go test ./internal/controller/security/ -v` |
| E2E (Kind) | `make test-e2e` — builds image, creates Kind cluster, deploys, runs `test/e2e/` with `-tags=e2e`, cleans up |
| Run locally | `make run` (needs Vault env vars + `operator.jwt`, see below) |
| Helm deploy | `make helm-deploy IMG=<registry>/<project>:<tag>` |

## Running locally

The operator uses Kubernetes ServiceAccount JWT (no static tokens). To run outside a cluster:

```bash
kubectl create token vaultweaver-operator -n vaultweaver-system --audience vault > operator.jwt
export VAULT_ADDR=http://127.0.0.1:8200
export VAULT_AUTH_KUBERNETES_ROLE=vaultweaver-operator
export VAULT_AUTH_KUBERNETES_JWT_PATH="$(pwd)/operator.jwt"
make run
```

Required env vars: `VAULT_ADDR`, `VAULT_AUTH_KUBERNETES_ROLE`. Optional: `VAULT_AUTH_KUBERNETES_MOUNT` (default `kubernetes`), `VAULT_AUTH_KUBERNETES_JWT_PATH` (default `/var/run/secrets/kubernetes.io/serviceaccount/token`).

## Testing quirks

- **Unit tests** use Ginkgo/Gomega with envtest (real K8s API + etcd). Suite: `internal/controller/security/suite_test.go`.
- **E2E tests** require an isolated Kind cluster. Build tag `e2e`; run with `make test-e2e`. Env overrides: `KUBECTL_KUBERC=true` (default off for isolation), `CERT_MANAGER_INSTALL_SKIP=true`.
- **Codegen** is not needed after every edit — only after changing type definitions or kubebuilder markers.
- Order when editing types: `make manifests && make generate && make lint-fix && make test`.

## Vault client

Defined as interface `vault.Client` in `internal/vault/client.go`. A hand-written `MockClient` lives in `internal/vault/mock.go` with function fields for each method — used in controller tests. No external mock framework.

## Lint

- `.golangci.yml` is v2 format. Enables ~20 linters including `ginkgolinter`, `logcheck` (via custom plugin from `.custom-gcl.yml`), `revive`, `lll`.
- Long-line exceptions: `api/*` and `internal/*`.
- `make lint-config` validates the linter config itself.

## Helm

Chart at `dist/chart/`. Built via `kubebuilder edit --plugins=helm/v2-alpha`. To regenerate after manifest changes: backup custom values, re-run `kubebuilder edit --plugins=helm/v2-alpha --force`, restore values.
